Overview

Active is an Easy-rated Windows machine on HackTheBox that simulates a real-world Windows Server 2008 R2 Domain Controller running Active Directory. Despite its "Easy" rating, it covers two highly relevant and frequently seen attack techniques in real penetration tests: Group Policy Preferences (GPP) credential exposure and Kerberoasting.

The attack chain begins with anonymous SMB enumeration to discover a readable Replication share, which leaks an encrypted password from a Groups.xml GPP file. After decrypting it with gpp-decrypt, we obtain credentials for the SVC_TGS service account. Using these credentials, we perform a Kerberoasting attack against the Administrator account, crack the resulting TGS ticket hash with hashcat, and achieve full domain compromise.

01 ——

Reconnaissance — Nmap

We begin with a full service version scan against the target to identify open ports and determine the attack surface.

bash nmap scan 
nmap -sC -sV 10.129.5.184

Key findings from the scan:

Classic Domain Controller fingerprint

Ports 53 (DNS), 88 (Kerberos), 389/3268 (LDAP), 445 (SMB), 464 (kpasswd) all open — confirming this is a Windows Active Directory Domain Controller. LDAP banner leaks the domain name: active.htb. OS is Windows Server 2008 R2 SP1.

SMB Signing Enabled

SMB2 security mode shows message signing is required — relay attacks (SMB relay / NTLM relay) are not viable here. We pivot to other attack surfaces.

02 ——

SMB Enumeration — Anonymous Access

With the domain identified, we use smbmap to enumerate available SMB shares using a null/anonymous session.

bash smbmap anonymous 
smbmap -H 10.129.5.184
🔍 Replication Share — READ ONLY (No Auth Required)

While ADMIN$, C$, IPC$, NETLOGON, SYSVOL, and Users all require authentication, the Replication share is accessible anonymously with read-only permissions. This is a misconfigured SYSVOL replica — a goldmine for GPP attacks.

03 ——

Exploring the Replication Share

We recursively enumerate the Replication share to map out its directory structure and look for sensitive files.

bash recursive share enumeration 
smbmap -H 10.129.5.184 -r Replication --depth 10

The share mirrors a SYSVOL structure containing Active Directory Group Policies. Navigating the folder tree, we find the critical file at:

04 ——

GPP Password Decryption

We download and read Groups.xml. This file was created by a Group Policy Preferences task that sets a local user account password — a practice Microsoft deprecated in MS14-025, but the encrypted credential remains in this old deployment.

xml Groups.xml content 
<?xml version="1.0" encoding="utf-8"?>
<Groups>
  <User name="active.htb\SVC_TGS"
        cpassword="edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeX..."
        userName="active.htb\SVC_TGS" />
</Groups>

The cpassword field is AES-256 encrypted — but Microsoft publicly documented the static decryption key in their MSDN documentation (MS14-025). The tool gpp-decrypt uses this known key to decrypt it instantly:

bash GPP decryption 
gpp-decrypt "edBSHOwhZLTjt/QS9FeIcJ83mjWA98gw9guKOhJOdcqh+ZGMeXOsQbCpZ3xUjTLfCuNH8pG5aSVYdYw/NglVmQ"

05 ——

Kerberoasting — TGS Request

With valid domain credentials, we can perform Kerberoasting. This attack requests Kerberos TGS tickets for accounts with registered Service Principal Names (SPNs). The tickets are encrypted with the service account's NTLM hash and can be cracked offline.

bash Impacket GetUserSPNs 
GetUserSPNs.py active.htb/SVC_TGS:GPPstillStandingStrong2k18 -request -outputfile hashes.txt
⚡ SPN Found on Administrator Account

The Administrator account has the SPN active/CIFS:445 registered. This is highly unusual and dangerous — a domain admin account acting as a service account means cracking its TGS ticket gives us domain administrator access.

Impacket outputs the TGS-REP hash in hashcat-compatible format to hashes.txt:

hash Kerberos 5 TGS-REP hash (truncated) 
$krb5tgs$23$*Administrator$ACTIVE.HTB$active.htb/Administrator*
$58e7e32376b07691c6b9960eea23d89a$30af5d0b2bef...[truncated]

We use hashcat with mode 13100 (Kerberos 5 TGS-REP) and the rockyou.txt wordlist to crack the hash offline.

Key Takeaways & Mitigations

1. GPP Passwords Are Never Safe (MS14-025)

Microsoft published the AES key used to encrypt GPP passwords. Any cpassword field in SYSVOL/Group Policy XML files can be trivially decrypted. Mitigation: Remove all GPP-stored passwords. Use LAPS (Local Administrator Password Solution) for local account management instead.

2. Kerberoastable Service Accounts With Weak Passwords

Any domain user can request TGS tickets for SPN-registered accounts. If those accounts have weak passwords, they will be cracked. Mitigation: Use long (25+ char) random passwords on all service accounts. Use Group Managed Service Accounts (gMSA) which auto-rotate. Regularly audit SPNs — especially on high-privilege accounts.

3. Domain Admins Should Never Have SPNs

The Administrator account had an SPN registered, making it Kerberoastable. Domain admin accounts should never double as service accounts. Mitigation: Segregate privileged accounts from service accounts. Monitor and alert on SPN assignments to privileged users.

4. SYSVOL/Replication Share Permissions

The Replication share was world-readable (anonymous access). While SYSVOL is typically readable by all domain users, anonymous access to a SYSVOL replica is a configuration error. Mitigation: Audit SMB share permissions and ensure anonymous access is disabled.
